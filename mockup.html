Brick Oracle - Product UI Mockup

Screen-level mockup built from current user stories and issues. Focus is user input, expected output, and UI states for demo readiness.

#3 Search Bar #9 Set Viewer #5 Collection Store #8 Collection Viewer #4 Upload Collection #10 Missing Piece Finder
MVP flow: Search -> Set Detail -> Add to Collection -> Collection View
Screen A. Search Page
#3 Search Bar
Top Bar: logo, nav, quick link to collection
Main Input: keyword/set number text field + Search button
Result List: set image, set id, set name, piece count, "View Set" button

Inputs

  • query text (set name or set number)
  • optional sort option (relevance, piece count)

Outputs

  • search result cards with matched sets
  • result count and query summary
Success: relevant set cards displayed.
Empty Query: "Enter a set name or number".
No Match: "No sets found" + refine tips.
Screen B. Set Detail Page
#9 Set Viewer
Set Header: image, set name, set id, theme, piece count
Actions: Add to Collection, Compare with My Inventory
Part Summary: top required parts and colors

Inputs

  • set id from search results
  • user click on "Add to Collection"

Outputs

  • set metadata and inventory summary
  • confirmation toast after add action
Success: set detail loaded and actions enabled.
Partial Data: missing image fallback shown.
Load Error: retry button + error text.
Screen C. Collection Page
#5 Collection Store #8 Collection Viewer
Collection Stats: total sets, total pieces, recent adds
My Sets Table: set id, name, owned qty, remove action
Insight Widget: "Closest buildable sets" preview

Inputs

  • add/remove set actions
  • filter by theme or piece range

Outputs

  • updated set list and totals
  • persistence confirmation in collection store
Success: collection saved and refreshed.
Empty Collection: CTA to search sets.
Save Error: local retry + warning banner.
Screen D-1. Collection Input (Default)
#5 Collection Store
Quick Add Form: add by set number or part number with quantity
Input Helpers: inline validation, recent additions, duplicate warning
Collection Update Feed: latest added sets/parts with undo action

Inputs

  • manual set/part entry with quantity
  • quick add submit and undo actions

Outputs

  • collection update summary for manual entries
  • persistent collection records in store
Success: item added to collection immediately.
Warning: duplicate entry, ask to merge quantity.
Error: invalid set/part id format.
Screen D-2. Advanced Import + Missing Piece Finder
#4 Upload Collection #10 Missing Piece Finder #7 Brick Diff
Bulk Import (Advanced): upload CSV + template download + format guide
Validation Panel: parsed rows, invalid rows, fix hints, manual correction action
Comparison Output: missing piece list by target set + completion percent

Inputs

  • CSV inventory file
  • target set selection for comparison

Outputs

  • import summary (accepted/rejected rows)
  • missing pieces table and build completion score
Success: import completed and missing-piece diff generated.
Validation Warning: malformed import rows with suggested fixes.
Format Error: invalid CSV schema message.
Complexity Notes (D-2):
  • CSV schemas differ by source, so parser normalization is required.
  • Part ID and color mapping must be reliable before missing-piece diff is trustworthy.
  • Large uploads need validation performance and clear row-level error reporting.
  • Diff logic must handle duplicate rows and quantity aggregation correctly.
